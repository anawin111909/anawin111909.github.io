Project: Game QA Testing Workflow

A structured QA testing process applied to a Unity puzzle game, covering gameplay verification, test case execution, and bug tracking. The workflow demonstrates how issues are identified, documented, and communicated during the testing cycle.

  • Feature-Based Test Checklist – Verified core gameplay systems including character movement, camera control, animations, VFX, and player interactions.
  • Manual Test Case Execution – Designed structured test cases with clear pre-conditions, test steps, expected results, and recorded pass/fail outcomes.
  • Bug Reporting & Tracking – Documented issues with reproduction steps, severity levels, screenshots, and tracked them using a board-based workflow.

Tools: Unity • Manual Testing • QA Documentation • Bug Tracking

Dissertation: AI Game Testing Tools Review

A research project exploring how Artificial Intelligence can automate game testing workflows. The study focused on training an AI agent in Unity ML-Agents to automatically explore game environments and detect potential gameplay bugs.

  • AI agent trained using Unity ML-Agents (PPO)
  • Automated detection of gameplay bugs (collision, clipping, unreachable areas)
  • Procedural test arena for automated exploration
  • Custom reward system to guide bug-finding behaviour
  • Integration between Unity and Python for training pipeline

Tech: Unity • C# • Python • ML-Agents • Reinforcement Learning

Role: AI System Developer / Researcher
